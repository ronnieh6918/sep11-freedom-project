# Presentation Plan

## Hook
* Six chances to prove you're the smartest in the group chat.
* Interactive and educational, Wordle delivers daily entertainment while actively sharpening your vocabulary and cognitive skills.

## Product
* Wordle Remix (6 letters)

## Process
* Flowchart of plan
  * MVP: Explain some of my early code, for example how I made multiple ReactJS components with checkGuess checking if my guess matches the secret word
  * Beyond MVP: Explaining how I made the tiles animated (flipped whenever user guesses)
* Code snippets of:
  * MVP: Explain what state and props are in ReactJS (essential for all the function inputs)
  * Beyond MVP: CSS code and my Javascript code

## Conclusion
* URL to project: [SEP11 Freedom Project](https://ronnieh6918.github.io/sep11-freedom-project/)
* Takeaways
  * Dedicate time during each learning log to learn your software. Developing your prototype early grants you a massive leap for final improvements (going beyond MVP)
  * Present your prototype to peers collectively. Gathering diverse feedback early allows you to fix critical flaws before your public launch.
  * Making use of what you have. If you try to add all the possible feedbacks, you would not have enough time trying to make your software perfect, but what really matters is to always make sure your product works before adding onto whatever else is necessary.
